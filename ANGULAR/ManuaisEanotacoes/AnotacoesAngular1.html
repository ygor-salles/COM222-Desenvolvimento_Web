1 - Cada component no Angular possui tres arquivos HTML, CSS, TS. E possui uma cadeia de componentes superior e/ou infeiroir na arvore de componentes ---------------------------------------------------------------------------------------------------------- 2 - Os elementos dentro do componente são acessados somente dentro dele mesmo. Ex: uma classe CSS só será aplicada somente dentro do HTML na qual fazem parte no componente. Tipo, componet cadastro ---------------------------------------------------------------------------------------------------------- 3 - DECORATOR => Padrão de projeto que tem como objetivo evitar herança, ao inves de herança trabalha com composição para extender um determinado objeto. o decorator pode ser um component @Component, diretiva @Directive, ... dentro do angular *Para um Atribute Directives => permite aplicar um atributo personalizado de seletor CSS por exemplo ---------------------------------------------------------------------------------------------------------- 4 - STRUCTURAL => Adiciona o remove elemento da DOM no projeto via TS que será transpilado para JS EX's:
===> irá exibir ou não o elemento do formulário SE o produto estiver definido ou setado, SENÃO irá remover o formulário
  • {{ product.name }}
==> irá adicionar novos elementos na DOM de acordo com a lista de produtos, através do *ngFor ---------------------------------------------------------------------------------------------------------- 5 - PROPRIETY BINDING => acesso do HTML através de um atributo no TS. O atributo modificado no TS altera no HTML (exibi no prompt), mais alterando do HTML não altera no TS, é Unidirecional Para Atributos usa [] EX: HTML [dataSource]="products"
TS products = Product[] ---------------------------------------------------------------------------------------------------------- 6 - EVENT BINDING => acesso do HTML através de um método no TS. Através de um evento disparado na Aplicação, executa um método no TS Para Métodos usa () EX: HTML TS createProduct(): void { ... } ---------------------------------------------------------------------------------------------------------- 7 - DATA BINDING *ONE WAY DATA BINDING: O que altera no TS altera no HTML, mais vice versa isso não ocorre. Utiliza [] EX: HTML TS nome: string *TWO WAY DATA BINDING: O que altera no TS altera no HTML, e vice versa. Utiliza [(ngModel)] EX: HTML TS nome: string ---------------------------------------------------------------------------------------------------------- 8 - ROTAS: Através do menu da aplicação ou da URL o usuário consegue navegar entre os componentes na aplicação através da rotas. EX: * Produtos (1 - Se o usuário clica nesse link) (2 - irá olhar na lista de rotas qual o path que bate com o link selecionado) * TS const routes = Routes [{ path: "products" component: ProductCrudComponent }, { path: "products/create" component: ProductCreateComponent }]; (3 - Após carregar o componente vair jogar pra dentro de router-outlet, onde fica o HTML princinpal do componente) * ---------------------------------------------------------------------------------------------------------- 9 - PIPES |: Interpolar o que tiver dentro de Double Mustach {{ }} EX:

{{ produto.vencimento | date }}

nesse exemplo irá aplicar uma formatação do método date em cima do atributo produto.vencimento

{{ product.price | currency: 'BRL' }}

nesse exemplo irá aplicar uma formatação do método currency passando como parametro 'BRL' em cima do atributo produto.vencimento *pode ter varias aplicações de métodos utilizando os PIPES (Encadeamento de PIPES)

O vencimento é {{ produeto.vencimento | date:'fullDate' | uppercase }}
